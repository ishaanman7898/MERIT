MERIT

Mass Email & Real-time Inventory Tracker — Setup Guide

0 / 5 steps completed
1
Fork & Clone the Repository

Fork this repository on GitHub, then clone it to your local machine.

git clone https://github.com/YOUR_USERNAME/merit.git
cd merit

After cloning, push to your fork's main branch. GitHub Actions will automatically deploy this setup page to your GitHub Pages.

2
Create a Supabase Project
  1. Go to supabase.com/dashboard and create a new project.
  2. Once created, go to Settings → API.
  3. Copy the Project URL — it looks like https://xyzcompany.supabase.co.
  4. Copy the anon / public key — it starts with eyJhbGciOi....

Save both values — you'll need them in Steps 3 and 5.

3
Run the Database Schema
  1. In your Supabase project, go to SQL Editor (left sidebar).
  2. Click New query.
  3. Open the file supabase/schema.sql from the repository and paste its entire contents into the editor.
  4. Click Run.

This creates all tables (products, inventory, email_campaigns, orders, config) and the inventory_summary view.

4
Deploy to Streamlit Community Cloud
  1. Go to share.streamlit.io and sign in with GitHub.
  2. Click New app.
  3. Select your forked merit repository.
  4. Set the Main file path to: app/main.py
  5. Click Deploy.

The app will install dependencies from requirements.txt and launch.

5
Complete First-Run Setup

When MERIT launches for the first time, it will show a setup wizard. Fill in:

  • Supabase URL and Anon Key — from Step 2.
  • SMTP Sender Email — your Gmail address (e.g. firm@gmail.com).
  • SMTP App Password — generate one at Google App Passwords. (Requires 2FA enabled on your Google account.)
  • Firm Name — used in email headers and the "From" field.
  • Logo URL (optional) — a direct link to your firm's logo image. If broken or empty, emails gracefully fall back to the firm name as text.
  • Accent Colors — navy and gold defaults match the VEI brand.

Click Save & Configure, then restart the app. MERIT is now fully operational.

6
Upload Product Images

After setup, go to Product Images in the sidebar to upload product photos:

  • Images are automatically compressed (resized to max 1200px, JPEG quality 80).
  • Stored in Supabase Storage (a "product-images" bucket is auto-created).
  • When you send order emails, these images are automatically attached.
  • Supports 2 images per product (primary + secondary).
  • Bulk upload — name files starting with the SKU (e.g., SKU123_front.jpg).

If the auto-created bucket doesn't work, go to Storage in your Supabase dashboard and create a bucket named product-images with public access.
